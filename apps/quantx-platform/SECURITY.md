# Security Policy

## Supported Versions

| Version | Supported | MDS Reference |
|---------|-----------|---------------|
| 1.x     | Yes       | Section 26 |

## Reporting a Vulnerability

Per MDS Section 32 (Dependency Management) and Section 33 (Secrets Management):

1. **Do not** open a public issue for security vulnerabilities
2. Email security@quantx.ai with vulnerability details
3. Include reproduction steps and potential impact assessment
4. Response within 24 hours per MDS Section 27 SLA

## Security Standards

This project follows:
- OWASP ASVS (MDS Section 30)
- NIST Secure SDLC (MDS Section 30)
- ISO/IEC 27001 (MDS Section 30)

## Compliance

Security artifacts must comply with:
- Sharia Mode requirements (MDS Section 30)
- GDPR data protection (MDS Section 30)
- SOC 2 controls (MDS Section 30)
